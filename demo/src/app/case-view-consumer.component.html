Case View component

This component is used to view a case. The elements of the contract are:

  • Input fields:
    • case - an id of a case (e.g. 1111222233334444)
    • hasPrint - optionally display/hide print button (true by default)
    • hasEventSelector - optionally display/hide event selector dropdown (true by default)
    • error - http error as a result of failed navigation (e.g. callback warnings or errors); it is required to display errors correctly inside case viewer
  • Output fields:
    • navigationTriggered - notification about triggered navigation (current possible navigations below)
      • DRAFT_DELETED - navigation after a sucessful deletion of a draft
      • ERROR_DELETING_DRAFT - navigation after an error occurs on trying to delete draft
      • DRAFT_RESUMED - navigation after a draft is resumed
      • EVENT_TRIGGERED - navigation after event is triggered
      • NO_READ_ACCESS_REDIRECTION - navigation after user has no READ access when redirected to case view after a case created
